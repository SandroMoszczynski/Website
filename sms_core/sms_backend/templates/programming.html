Python

I started learning Python in my second year of my BSc in 2012. Since then I have used it for many years with varying degrees of expertise. During my BSc I became familiar with Numpy and Matplotlib in regular use for my second and third year laboratory projects, although this was limited in comparison to my current standard of coding.


During my time at UCL I used Python in most of my classes and for my MSc project. I learned how to parallelise my code to multiple CPU cores and even how one would go about sending off some calculations to a GPU. In addition to this I learned some Tensorflow and learned a lot about how Python can interact with running processes on Windows and Linux and how to manipulate these, this was most applicable to my MSc project as that mostly ran through command line. Other topics I learned include how to create a library that would be available to download through Python and the use of Pandas, Numba and OpenCL. I also made extensive use of Jupyter notebooks, as can be seen in my GitHub links below.


During this time I was taught how to use GitHub, which I have continued to use for both this website and other pesonal projects. It was also extensively used unit testing, with 2 of the projects listed below making use of Continuous Integration through Travis and GitHub.


While the class functionality of Python is not inherent to it, I have made use of OOP with Python in both my project and through classwork. However due to its slightly clunky implementation I only use them for projects where keeping a neat and organised flow of functions is necessary.


Python to me is the epitome of an easy to use programming solution. With a few commands I can open up a Jupyter Notebook and quickly test whatever small piece of code I just realised I need make work. Even for larger projects with tools like Numba and OOP to keep organised one can create fast and neat code that runs almost as quickly as a compiled language.
